## [v0.2.0] - 2026-02-08

### Highlights

- Rename to schemalint and add scale profile for strict static type compatibility

### Breaking

- Rename binary from `schemago` to `schemalint` ([`a96efd5`](https://github.com/grokify/schemakit/commit/a96efd5))
- Rename module from `github.com/grokify/schemago` to `github.com/grokify/schemalint` ([`a96efd5`](https://github.com/grokify/schemakit/commit/a96efd5))

### Added

- `--profile` / `-p` flag to select linting profile (`default`, `scale`) ([`c0fed40`](https://github.com/grokify/schemakit/commit/c0fed40))
- Scale profile with `composition-disallowed` rule: disallow `anyOf`, `oneOf`, `allOf` ([`c0fed40`](https://github.com/grokify/schemakit/commit/c0fed40))
- Scale profile with `additional-properties-disallowed` rule: disallow `additionalProperties: true` ([`c0fed40`](https://github.com/grokify/schemakit/commit/c0fed40))
- Scale profile with `missing-type` rule: require explicit `type` field ([`c0fed40`](https://github.com/grokify/schemakit/commit/c0fed40))
- Scale profile with `mixed-type-disallowed` rule: disallow type arrays like `["string", "number"]` ([`c0fed40`](https://github.com/grokify/schemakit/commit/c0fed40))
- Goreleaser configuration for cross-platform builds and Homebrew tap ([`b5d1fab`](https://github.com/grokify/schemakit/commit/b5d1fab))
- `--property-case` flag to enforce property naming conventions (e.g., camelCase, snake_case) ([`73ed44f`](https://github.com/grokify/schemakit/commit/73ed44f))

### Changed

- Update CLI description to emphasize static type compatibility for all languages (Go, Rust, TypeScript, etc.) ([`a96efd5`](https://github.com/grokify/schemakit/commit/a96efd5))

### Documentation

- README with scale profile docs and Homebrew installation ([`3d4a088`](https://github.com/grokify/schemakit/commit/3d4a088))
